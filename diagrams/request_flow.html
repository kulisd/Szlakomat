Przepływ żądania — HTTP → Domain → Result

POST /api/products — command DefineProductType wędruje przez warstwy aplikacji Szlakomat

Railway-oriented

  • • Handlery zwracają Result<string, T>
  • • Typ błędu = komunikat (string)
  • • SuccessOf / FailureOf — dwa tory
  • • Kontroler dopasowuje: IsSuccess() / GetFailure()
  • • Brak rzucania wyjątków dla błędów biznesowych

Konwencje CQRS

  • • Command/Query: IRequest<TResponse>
  • • Handler: sealed, ta sama przestrzeń nazw co komenda
  • • Jedna odpowiedzialność handlera
  • • Dispatcher: MediatR ISender.Send()
  • • Wymuszone przez Architecture.Tests

Warstwy w locie

  • • ProductsController → Api
  • • DefineProductType + Handler → Application
  • • ProductBuilder, IProductTypeRepository → Domain
  • • InMemoryProductTypeRepo → Infrastructure
  • • Kierunek zależności: Api → App → Domain ← Infra
