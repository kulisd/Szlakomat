Bounded Contexts — Szlakomat.Products

Cztery konteksty domenowe i shared kernel — struktura projektu Szlakomat.Products.Domain

Cztery konteksty

  • • Catalog — definiuje CZYM produkt jest; typy niezmienialne
  • • CommercialOffer — definiuje KIEDY i JAK produkt jest prezentowany
  • • Instances — konkretne realizacje typów z egzekucją ograniczeń
  • • Relationships — skierowane relacje między typami katalogowymi

Wzorzec Composite — dwa poziomy

  • • IProduct — leaf: ProductType, composite: PackageType
  • • IInstance — leaf: ProductInstance, composite: PackageInstance
  • • Symetria pomiędzy poziomem typu (Catalog) a poziomem realizacji (Instances)
  • • Zachowuj równoległość przy rozszerzaniu modelu

Shared Kernel

  • • IProduct, IInstance — abstrakcje Composite
  • • IProductIdentifier, ISerialNumber — identyfikacja
  • • Result<TF, TS> — railway-oriented error handling
  • • Quantity, Unit — domena ilości i jednostek
  • • IApplicabilityConstraint — komponowalne specyfikacje
  • • Kontrakty repozytoriów (interfejsy) widoczne dla wszystkich kontekstów
