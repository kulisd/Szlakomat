Szlakomat.Products — Architektura warstwowa

Clean Architecture · kierunek zależności wymuszony przez testy NetArchTest

Kierunek zależności

  • • Domain nie odwołuje się do Application / Infrastructure / Api / MediatR
  • • Application zależy wyłącznie od Domain; bez Infrastructure / Api / ASP.NET Core
  • • Infrastructure nie może odwoływać się do Api
  • • Reguły wymuszane przez testy w LayerDependency/
  • • Api tłumaczy HTTP na żądania MediatR i dopasowuje Result<TF, TS>

Bounded contexty

  • • Catalog — ProductType / PackageType, niezmienialne po utworzeniu
  • • CommercialOffer — CatalogEntry + Validity (widoczność, nie blokuje instancji)
  • • Instances — ProductInstance, PackageInstance, Batch; tracking strategy
  • • Relationships — skierowane relacje między typami (DefiningPolicy)
  • • Shared kernel — Common/, Quantity/ (IProduct, IInstance, Result, Unit)

Testy architektoniczne

  • • NetArchTest.Rules — fitness functions na assembly
  • • LayerDependency — kierunek zależności warstw
  • • CqrsConventionTests — handlery sealed, IRequest<T>, jedna odpowiedzialność
  • • Immutability — value objecty bez publicznych setterów
  • • Visibility — agregaty internal; dostęp przez InternalsVisibleTo
  • • NamingConventions — wzorce nazw w warstwach
